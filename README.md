# MyIRS

## Project Overview

**MyIRS** is a private internal mobile application developed for an FMCG company.  
This application is used to support daily operational activities for multiple staff roles, with a focus on process efficiency, data accuracy, and operational reporting.

This application is intended for **internal use only** and is not publicly available.

---

## Core Features (High-Level)

- Daily **loading and unloading (bongkar muat)** feature  
- Daily **Point of Sales**  
- **Stock opname** and inventory-related features  
- **Permission feature** for entering, exiting, and moving within the company area  
- Other internal features to support FMCG operational staff  

Detailed workflows and business logic are not exposed as they are internal.

---

## Technical Stack

- **Mobile**: Flutter  
- **Backend**: Laravel  
- **Architecture**: Modular structure with focus on maintainability  
- **API Communication**: REST API

## Role & Responsibilities

My responsibilities in this project include:

- Analyzing user requirements from various operational staff  
- Translating user requirements into **UI/UX design**  
- Implementing features and performing **code review**  
- Coordinating feature delivery with **QA / tester**  
- Involved in application **deployment** process to PlayStore
- Providing **user training** and usage guidance  

---

## Key Learnings

- Applications used daily for operations must be fast, clear, and stable  
- Clean code and consistent project structure are important for long-term maintenance  
- Good communication between user, developer, and tester improves product quality  
- Deployment and post-release monitoring are important to keep operational activities running smoothly
